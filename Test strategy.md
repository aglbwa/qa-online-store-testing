# Стратегия тестирования

В рамках проекта используются следующие виды тестирования.

## Функциональное тестирование

Проверка соответствия функциональности требованиям.

## Smoke Testing

Проверка работоспособности основных функций приложения после запуска.

## Регрессионное тестирование

Повторная проверка ранее протестированного функционала после внесения изменений.

## UI-тестирование

Проверка корректности отображения элементов интерфейса.

## Исследовательское тестирование

Свободное тестирование приложения без заранее подготовленных сценариев.

## Тестирование совместимости

Проверка работы приложения в разных браузерах и при различных разрешениях экрана.

## Негативное тестирование

Проверка реакции системы на некорректные действия пользователя.

## Тестирование граничных значений

Проверка работы приложения при вводе минимальных, максимальных и пограничных значений.
